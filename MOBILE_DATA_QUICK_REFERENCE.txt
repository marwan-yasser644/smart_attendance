# ✅ Mobile Data Access - Quick Reference

## What Was Done

The system is now **fully configured** for students to submit attendance using mobile data (not just campus Wi-Fi).

### How It Works

1. **Lecturer generates QR code** in the attendance session
2. **System checks `BASE_URL` environment variable:**
   - ✅ If set → QR code points to college's public server
   - ⚠️ If not set → QR code points to local network only

3. **Student scans QR code:**
   - On campus Wi-Fi: Works ✅
   - On mobile data: Works if `BASE_URL` is set ✅

---

## For College IT: One-Line Setup

### Option 1: Public IP Address
```bash
export BASE_URL=http://192.168.1.50:5000
gunicorn --workers 4 --bind 0.0.0.0:5000 app:app
```

### Option 2: College Domain
```bash
export BASE_URL=https://attendance.college.edu
gunicorn --workers 4 --bind 0.0.0.0:5000 app:app
```

### Option 3: ngrok Testing
```bash
export BASE_URL=https://abc123.ngrok.io
python run.py
```

**That's it!** Students can now scan QR codes and submit attendance from anywhere with internet.
